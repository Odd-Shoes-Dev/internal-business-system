# Whop Payment Integration Configuration

Add these environment variables to your `.env.local` file to enable Whop payment processing:

## Required Variables

```bash
# Whop API Configuration
NEXT_PUBLIC_WHOP_COMPANY_ID=biz_xxxxxxxxxxxxx  # Your Whop company ID
WHOP_API_KEY=xxxxxxxxxxxx                      # Your Whop API key (for backend)

# Support Contact Information (shown for high-value plans > $2500)
NEXT_PUBLIC_SUPPORT_EMAIL=support@yourdomain.com
NEXT_PUBLIC_DISPLAY_PHONE=+1-800-XXXX-XXXX
NEXT_PUBLIC_WHATSAPP_LINK=https://wa.me/1234567890
```

## How It Works

### 1. **Plan Selection Flow**
- User signs up → Redirected to `/signup/select-plan`
- Selects region + subscription tier + billing period
- System looks up corresponding Whop plan ID from generated config

### 2. **Checkout Processing**
- **For plans under $2500**: Redirect directly to Whop checkout link
- **For plans over $2500**: Show contact form with WhatsApp/Email links

### 3. **After Successful Payment**
- Whop redirects back to `/checkout/success`
- User can access dashboard

## Plan IDs Location

Generated plan IDs are stored in:
- `src/lib/whop-config.generated.json` (raw data)
- `src/lib/whop-config.ts` (TypeScript exports with helper functions)

These are auto-generated by the GitHub Actions workflow (`setup-whop-products.ts`)

## Regenerating Plans

To update pricing or add new plans:

1. Update `src/lib/regional-pricing.ts` with new prices
2. Go to GitHub Actions → **Setup Whop Products** → **Run workflow**
3. Script auto-generates new config files
4. Changes are auto-committed to `main`

## Testing

To test payment flow locally:

```bash
# 1. Create a test account via signup form
# 2. Select a plan in plan selection page
# 3. You'll be redirected to Whop checkout (test or live depending on API key)
# 4. After payment, redirect to /checkout/success
```

## Troubleshooting

**"Plan not available for this region"**
- Check `whop-config.ts` - plan ID might be missing
- Some plans > $2500 are intentionally skipped (contact support instead)

**Checkout not redirecting**
- Verify `NEXT_PUBLIC_WHOP_COMPANY_ID` is set correctly
- Check browser console for plan ID errors

**Contact form not showing**
- Ensure `NEXT_PUBLIC_SUPPORT_EMAIL` and `NEXT_PUBLIC_WHATSAPP_LINK` are set
- Enterprise annual plans should trigger the contact form
